# 팀 AI 컨벤션 (도구 중립)

## 핵심 원칙

- 전역보다 로컬 모듈을 우선한다.
- 함수는 가능한 작고 순수하게 유지한다.
- 외부 입력은 항상 nil 가능성을 처리한다.
- Love2D 프레임 업데이트 로직은 결정적으로 유지한다.

## AI 어시스턴트 출력 규칙

- 변경 파일을 먼저 제시한다.
- 동작 변화와 리스크를 함께 설명한다.
- 최소 검증 체크리스트를 포함한다.
- 학생마다 환경이 다를 수 있으므로, 검증 방법은 task 우선으로 쓰고 OS별 예외는 분리한다.
- 검증은 테스트 하네스 우선으로 수행하고, 게임 전체 실행 검증은 필요한 경우 최소 횟수로만 수행한다.

## 총괄 에이전트 운영 규칙

- 세션 시작 시 `game-director`를 먼저 호출해 오늘의 Top 3 우선순위를 확정한다.
- 총괄 제안은 `핵심 가치 적합성 + 재미 임팩트 + 구현비용` 기준으로 정렬한다.
- 세션 종료 전 worklog(일일 1파일)에 `결정/변경/다음 TODO`를 반드시 누적 기록한다.

## Lua/Love2D 코딩 표준

- Love2D 프로젝트에서는 LuaJIT/Lua 5.1 호환 문법을 사용한다.
- 암묵적 전역 변수에 의존하지 않는다.
- update 와 draw 책임을 명확히 분리한다.
- 유틸리티 모듈에는 부수 효과를 넣지 않는다.
- 실행 경로나 런타임 가정은 문서와 task에 분리해서 기록한다.
