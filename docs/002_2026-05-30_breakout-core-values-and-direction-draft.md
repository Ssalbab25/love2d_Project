# 002 - 2026-05-30 - Breakout Core Values and Direction (Draft)

## 문서 목적

- 기존 벽돌깨기와의 차별성을 명확히 정의한다.
- 재미를 유지하는 핵심 루프와 개발 우선순위를 합의한다.
- 후속 기능 개발 시 판단 기준으로 사용할 공통 지침을 만든다.

## 핵심 가치

### 1. Skill Expressiveness
- 정의: 플레이어의 선택과 숙련이 결과를 분명히 바꿔야 한다.
- 적용 기준:
  - 같은 레벨에서도 고득점 경로가 다수 존재해야 한다.
  - 반사 각도, 리스크 선택, 체인 유지가 점수와 생존을 동시에 좌우해야 한다.

### 2. Fast Readability
- 정의: 세로 화면에서 0.3초 내 상황 판단이 가능해야 한다.
- 적용 기준:
  - 위협/기회/보상 UI가 명확히 분리되어야 한다.
  - 벽돌 체력, 배수 상태, 실패 리스크 표시가 가려지지 않아야 한다.

### 3. Meaningful Risk-Reward
- 정의: 안전 플레이와 고위험 플레이의 기대값이 다르게 설계되어야 한다.
- 적용 기준:
  - 체인 유지, 위험 구역 진입, 고체력 벽돌 우선 타격에 보상 구조를 부여한다.
  - 리스크는 즉시 체감되고, 보상은 숫자와 이펙트로 즉시 피드백한다.

### 4. Deterministic Core, Experimental Modes
- 정의: 코어 루프는 결정적으로 유지하되 모드는 빠르게 실험 가능해야 한다.
- 적용 기준:
  - 물리/충돌은 안정성을 우선한다.
  - 모드 규칙(체인, 타임어택, 서바이벌)은 별도 인터페이스로 주입한다.

## 차별화 콘셉트 제안

### A안: Combo Rhythm Breakout
- 판타지: 연속 파괴 리듬을 타며 화면을 지배한다.
- 차별점: "얼마나 빨리"가 아닌 "얼마나 끊기지 않게"를 핵심 목표로 둔다.
- 루프:
  - 30초: 체인 시작과 유지 감각 학습
  - 3분: 배수 유지/갱신 의사결정
  - 20분: 고난도 패턴에서 체인 최적화

### B안: Risk Lane Breakout
- 판타지: 위험 레인을 활용해 고보상을 노린다.
- 차별점: 특정 구역 타격 시 고점수, 실패 시 큰 패널티.
- 루프:
  - 30초: 위험 구역 규칙 이해
  - 3분: 안정과 고점 사이 스위칭
  - 20분: 레벨별 위험 맵 완전 학습

### C안: Draft Modifier Breakout
- 판타지: 매 스테이지 규칙을 선택해 나만의 빌드를 만든다.
- 차별점: 라운드 시작 전 모디파이어 선택으로 판이 달라진다.
- 루프:
  - 30초: 선택-결과의 즉시 체감
  - 3분: 시너지 탐색
  - 20분: 누적 선택 기반 장기 플랜

## 최종 제안

- 선택: A안 (Combo Rhythm Breakout)
- 선택 이유:
  1. 현재 코드 구조에 가장 저비용으로 붙일 수 있다.
  2. 손맛 차별점이 즉시 체감된다.
  3. 세로 화면에서 정보 전달이 단순하다.
  4. 향후 모디파이어/모드 확장과 궁합이 좋다.
  5. 하네스 우선 검증 전략과도 잘 맞는다.

## 시스템 설계 원칙

### 점수 구조
- 기본 점수: 벽돌 체력/난이도 기반.
- 배수 점수: 체인 타이머 내 연속 파괴 시 증가.
- 리셋 조건: 일정 시간 무파괴 또는 목숨 손실.

### 조작/손맛
- 패들 조작 응답성은 항상 고정 품질 유지.
- 반사각 제어는 의도적으로 학습 가능해야 한다.
- 피드백은 사운드, 미세 흔들림, 점수 팝업으로 일관화한다.

### 세로 UI
- 상단: 점수/목숨/배수
- 중앙: 핵심 플레이 영역
- 하단: 입력/상태 안내
- 텍스트보다 색/애니메이션으로 상태를 우선 전달한다.

### 온보딩 (첫 60초)
- 0~15초: 발사/이동만 학습
- 15~35초: 체인 개념 노출
- 35~60초: 체인 끊김 리스크와 회복 방법 노출

## 구현 로드맵

### Phase 1: MVP 차별화
- 체인 카운트 + 배수 점수 + 체인 HUD
- 완료 기준:
  - 체인 시작/유지/끊김이 명확히 보인다.
  - 하네스 테스트로 점수 계산 검증 가능

### Phase 2: 반복 플레이 강화
- 체인 유지 보너스(짧은 버프 또는 피버)
- 레벨별 체인 친화 배치
- 완료 기준:
  - 3판 이상 반복 시 전략 변화가 관찰된다.

### Phase 3: 모드 확장 준비
- mode 인터페이스 분리
- classic/combo 모드 선택 구조
- 완료 기준:
  - 코어 물리 수정 없이 모드 교체 가능

## 리스크와 가드레일

### 리스크
1. 이펙트 과잉으로 가독성 저하
2. 배수 설계 실패로 단조로운 최적해 고정
3. 터치 조작 미완성으로 모바일 체감 저하
4. 모드 확장 시 상태 분기 폭발
5. 실행 검증 과다로 개발 속도 저하

### 가드레일
- 테스트 하네스 우선 검증
- 전체 실행 검증은 입력/렌더 변경 시 최소 횟수만 수행
- 상태 전이는 단일 경로에서 관리
- 모드 규칙은 코어 물리와 분리
- worklogs에 매 세션 의사결정 근거 기록
