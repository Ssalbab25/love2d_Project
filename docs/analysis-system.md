# 체스 AI 및 형세 연산 시스템 명세서 (analysis-system.md)

본 문서는 플레이어의 vs AI 대국 및 게임 연산 백엔드 구조에 대한 명세서입니다. 기존 기획 중 실시간 그래픽 UI(평가 점수 바, 추천 다음 수 화살표)는 LÖVE2D 단일 스레드 렌더링 루프에서의 CPU 지연 방지 및 세련되고 깔끔한 클래식 체스 화면 구성을 위해 배제되었으며, 연산 로직은 AI 엔진 및 테스트 하네스를 위한 기능으로 고도화되었습니다.

---

## 1. 시스템 개요 (System Overview)

- **백엔드 분석 통합**: `AnalysisFacade`를 통해 체스 게임 루프와 AI 동작 알고리즘 간의 분석 연산을 수행합니다.
- **최적화 중심 설계**: 화면에 매 프레임 화살표나 그래프를 그리는 오버헤드를 완전히 걷어내고, 백그라운드 탐색 연산 및 기물 위치 평가(PST)의 성능 최적화에 집중하여 프레임 드랍을 원천 차단했습니다.

---

## 2. 데이터 구조 및 데이터베이스 (BookDatabase)

마스터 기보 오프닝 조회를 위한 해시 테이블 구조입니다.
- **오프닝 북 데이터베이스**: 수백 가지 공식 마스터 기보를 FEN(Forsyth-Edwards Notation) 키값으로 캐싱하여 관리합니다.

```lua
local BookDatabase = {
    ["rnbqkbnr/pppppppp/8/8/8/8/PPPPPPPP/RNBQKBNR w KQkq - 0 1"] = {
        { move = "e2e4", weight = 450 },
        { move = "d2d4", weight = 380 }
    }
}
```

## 3. 핵심 아키텍처 설계 (SOLID 원칙 적용)

### 3.1. OpeningBookMatcher (SRP: 기보 매칭)
- **역할**: 현재 보드를 표준 FEN 문자열로 인코딩한 후, `BookDatabase`에 매칭하여 오프닝 단계에서 최적의 추천 수를 즉시 반환합니다.

### 3.2. EvaluationEngine (SRP: 형세 평가)
- **역할**: 오프닝 북을 벗어난 중후반전 상황에서 정밀 위치 테이블(PST)을 기반으로 현재 보드의 점수(센티폰 단위)를 측정합니다.
- **평가 공식**:
  $$\text{Score} = \sum (\text{기물 가치}) + \sum (\text{PST 위치 가치})$$

### 3.3. AnalysisFacade (DIP: 의존성 역전)
- **역할**: 오프닝 북 매처와 PST 평가 엔진을 결합하여 고수준 대국 분석 기능을 단일 인터페이스로 제공합니다. AI 난이도 검색 모듈 및 테스트 스케줄러가 이 파사드를 의존하여 행마 결정을 조율합니다.

## 4. UI/UX 표현 규칙 (UI Removal)

체스 본연의 클래식하고 고급스러운 대국 경험을 선사하기 위해 아래의 분석 그래픽 UI 요소를 렌더링 영역에서 완벽하게 제외했습니다.
1. **평가 점수 바(Evaluation Bar) 제거**:
   - 매 프레임 형세를 그리기 위해 CPU를 점유하는 렉 유발 요소를 배제하고, 사이드패널 영역에 체스 대국 시계(Time) 및 기물 정보만 온전히 집중시켰습니다.
2. **추천 화살표(Hint Arrow) 제거**:
   - 플레이어가 외부 힌트에 의존하지 않고 체스의 전략적 사고에만 온전히 집중하도록 보드 위에 노출되던 안내 화살표를 제거하였습니다.
