# 003 - 2026-05-30 - Mobile Real-Device Check Routine

## 문서 목적

- 데스크톱 하네스 통과와 모바일 실기 체감 사이의 갭을 줄인다.
- 세로 모바일 벽돌깨기에서 반드시 확인해야 할 항목을 고정한다.
- 세션마다 같은 기준으로 PASS/FAIL을 기록할 수 있게 한다.

## 적용 범위

- 대상 빌드: LÖVE2D 기반 세로 모드 프로젝트
- 대상 기능: 터치 입력, 패들 추적 감각, UI 가독성, 프레임 안정성
- 사용 시점: 입력/렌더/UI/모드 밸런스 변경 후

## 실행 전 준비

1. 최신 브랜치 상태 확인
- git status가 의도한 변경만 포함하는지 확인

2. 기본 하네스 통과 확인
- 입력, 모드, 레벨, 점수 관련 하네스를 먼저 PASS로 만든다.

3. 실기 대상 기기 기록
- 기기명, OS 버전, 화면 주사율(가능하면) 기록

## 실기 체크리스트 (필수)

### A. 입력 응답성

1. 최초 터치 반응
- 항목: 터치 시작 후 패들 반응이 즉시 시작되는가
- 기준: 지연이 체감되지 않아야 한다.

2. 드래그 추적 일관성
- 항목: 좌우 빠른 드래그 시 패들이 끊기지 않고 따라오는가
- 기준: 순간 점프/역방향 떨림이 없어야 한다.

3. 가장자리 제어
- 항목: 화면 좌/우 끝 근처에서 패들이 안정적으로 멈추는가
- 기준: 경계 진동, overshoot가 없어야 한다.

### B. 가시성/가독성

1. 손가락 가림
- 항목: 터치 중 볼/벽돌/HUD 핵심 정보가 과도하게 가려지는가
- 기준: 플레이 판단 정보가 항상 보인다.

2. 상태 텍스트 가독성
- 항목: serve/level clear/result 문구가 작은 화면에서 읽히는가
- 기준: 확대 없이 판독 가능해야 한다.

### C. 게임 루프 안정성

1. 프레임 안정성
- 항목: 파티클, 흔들림, 팝업이 많은 상황에서 끊김이 없는가
- 기준: 조작 품질 저하가 체감되지 않아야 한다.

2. 상태 전이 안정성
- 항목: pause/result/모드 전환 시 입력 잠김/중복 입력이 없는가
- 기준: 의도한 흐름으로만 전이한다.

### D. 모드 체감 차별화

1. classic vs combo_rush 차이
- 항목: 같은 플레이 시간에서 난이도/점수 템포 차이가 체감되는가
- 기준: 플레이어가 즉시 모드 성격을 구분할 수 있어야 한다.

## 권장 테스트 시나리오

1. 2분 드래그 집중 시나리오
- 목적: 터치 추적/경계 안정성 검증
- 방법: launch 후 공 생존을 유지하며 좌우 끝 왕복 드래그 반복

2. 3레벨 도달 시나리오 (combo_rush)
- 목적: 후반 속도/가독성/입력 부담 검증
- 방법: combo_rush 모드로 연속 진행하여 레벨 3 진입

3. 오버레이 전이 시나리오
- 목적: pause/result 씬 전환 안정성 검증
- 방법: 플레이 중 pause, 재개, 게임오버/클리어 후 재시작
