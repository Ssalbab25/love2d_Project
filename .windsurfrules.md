
# 에이전트 필수 워크플로우 규칙 (Agent Workflow Protocol)

당신은 코드를 작성하거나 수정할 때, 반드시 아래의 3단계 워크플로우를 예외 없이 따라야 합니다.

## 1. 계획 및 문서화 (Pre-coding)
- 새로운 기능을 추가하기 전, `docs/` 폴더 내의 설계 문서를 먼저 읽고 구조를 파악하십시오.
- 기존 아키텍처에 변경이 생길 경우, 코드 작성 전에 `docs/` 내의 관련 문서를 먼저 업데이트하십시오.

## 2. 테스트 주도 개발 (During-coding)
- `src/` 폴더에 새로운 모듈이나 클래스를 생성할 때, 반드시 `tests/` 폴더에 대응하는 테스트 스크립트를 먼저 혹은 동시에 작성하십시오. (예: `src/pieces/Pawn.lua` 생성 시 `tests/test_Pawn.lua` 생성)
- Lua의 내장 `assert()`를 활용하거나 프로젝트 내에 합의된 테스트 프레임워크를 사용하여 객체의 상태와 경계값을 검증하십시오.

## 3. 작업 일지 작성 (Post-coding / 완료 보고)
- 사용자에게 "작업을 완료했습니다"라고 보고하기 직전에, `worklogs/` 폴더에 오늘 날짜(예: `YYYY-MM-DD.md`)의 마크다운 파일을 생성하거나 열어서 아래 양식으로 기록을 남기십시오.
  - **Time**: (작업 완료 시간)
  - **Task**: (무엇을 만들었거나 수정했는지 요약)
  - **Files Modified**: (건드린 파일 목록)
  - **Test Result**: (작성한 테스트의 성공 여부)
  - **Next Step**: (다음에 이어서 해야 할 작업 제안)
