# 목적

이 스킬은 Lua 언어 자체를 설계, 구현, 디버깅, 리팩터링할 때 사용합니다.

## 절차

1. 문제의 Lua 언어 레벨 원인을 먼저 식별합니다.
2. 모듈 경계, 데이터 구조, 함수 책임을 정리합니다.
3. 암묵적 전역과 불필요한 부수 효과를 제거합니다.
4. 메타테이블, 클로저, 반복문, 에러 처리의 의도를 명확히 합니다.
5. 성능과 가독성의 균형을 고려한 수정안을 제시합니다.

## 출력 형식

- 현재 문제 요약
- Lua 수준의 원인
- 추천 수정안
- 경계 사례와 주의점
- 검증 단계
