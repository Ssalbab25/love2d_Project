# 목적

LÖVE2D + Lua 프로젝트의 전체 아키텍처 구조를 설계하거나 개선할 때 이 스킬을 호출합니다. 강한 의존성을 느슨하게 만들고, 모듈의 경계를 명확하게 정립하여 고성능 2D 게임의 토대를 만듭니다.

## 핵심 수행 절차

1. **의존성 맵 분석:** 현재 프로젝트 구조와 파일 간의 결합성 및 의존 경로를 꼼꼼하게 모델링합니다.
2. **단일 책임 원칙(SRP) 적용:** 각 모듈(카메라, 입력, 설정, 에셋 로더 등)이 오직 하나의 도메인 책임만 갖도록 재정렬합니다.
3. **update/draw 책임 한계 보장:** 게임 엔진 상태 루프가 꼬이지 않도록 업데이트 로직과 렌더링 호출을 완전히 분리합니다.
4. **의존성 주입(DI):** 전역 변수를 쓰지 않고, 상위 인스턴스 또는 매니저 객체에서 서브 시스템으로 명시적 생성자나 초기화 함수를 통해 필요한 모듈을 주입합니다.
5. **성능 프로파일 최적화 계획:** 빈번한 가비지 컬렉션(GC)이나 로컬 범위 이탈 등으로 인한 힙 오버헤드를 막는 리팩터링 전략을 동반합니다.

## 아키텍처 핵심 기준

- **최대 독립성:** 개별 모듈은 부수 효과 없이 단독 로드되어 동작을 테스트할 수 있어야 합니다.
- **게임 루프의 결정성:** 프레임율(FPS)의 흔들림 속에서도 상태 업데이트 보간이 안전하도록 dt 연산을 통합합니다.
- **느슨한 결합도(Loose Coupling):** 강결합된 종속 관계를 이벤트 기반 통신이나 콜백 구조로 전환하여 응집도를 끌어올립니다.

## 출력 양식 가이드라인

- **[구조적 진단]** 현재 모듈 아키텍처의 리스크 및 병목 지점
- **[아키텍처 변경안]** 모듈별 상세 개선 설계안(클래스/테이블 다이어그램 등 텍스트 구조 포함)
- **[리팩터링 로드맵]** 변경이 다른 서브 시스템을 깨뜨리지 않도록 하는 점진적 롤아웃 계획
- **[동작 보장/검증]** 테스트 시나리오 및 LÖVE2D 실행 태스크 안내
