# 목적

LÖVE2D 환경에서 신규 콘텐츠(게임플레이 스테이지, UI 컴포넌트, 물리 엔티티 등)를 생성하거나 기존 피처를 신속하게 뼈대화(Scaffolding) 및 통합하고자 할 때 이 스킬을 호출합니다.

## 핵심 수행 절차

1. **상태 및 영향도 정의:** 구현할 피처가 미치는 전역/로컬 게임 상태와 수정될 스크립트 파일을 정의합니다.
2. **최소 기능 세트(MVP) 설계:** 기존 게임 동작이 깨지지 않도록 하위 호환 구조를 다집니다.
3. **update/draw 라이프사이클 분할 구현:**
   - **`update(dt)` 계층:** 입력 처리, 물리 반응, 타이머 및 애니메이션 카운트 등의 상태 변경만 구현합니다.
   - **`draw()` 계층:** 렌더링 설정 복구(`love.graphics.push/pop`), 컬러 바인딩, 리소스 드로우 등의 화면 출력만 수행합니다.
4. **결합 인터페이스 최소화:** 메인 루프 또는 상태 매니저에서 이 피처를 임포트하여 로드하는 인터페이스 코드를 극도로 명료하게 제시합니다.

## 출력 양식 가이드라인

- **[신규/수정 파일]** 절대 경로와 함께 생성 또는 변경될 스크립트 목록
- **[동작 및 상태 전이 메커니즘]** 피처 활성화 시 게임 루프에 통합되는 핵심 로직 스니펫
- **[자가 검증 단계]** 작성한 신규 피처를 테스트하기 위한 단계별 재현 절차 및 태스크 실행 방법 안내
