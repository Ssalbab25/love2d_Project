# Antigravity 전용 Lua 및 LÖVE2D 코딩 규칙

이 규칙은 Antigravity 에이전트가 이 워크스페이스 내의 Lua 코드를 생성, 수정 및 리팩터링할 때 반드시 준수해야 하는 지침입니다.

## 1. LuaJIT 및 LÖVE2D 호환성
- 모든 Lua 스크립트는 **LuaJIT(Lua 5.1 호환)** 문법과 의미론을 철저히 유지해야 합니다.
- LÖVE2D의 최신 기능(11.x+)을 사용하되, 하위 호환성과 모듈식 확장을 저해하지 않는 안정적인 API 위주로 호출합니다.

## 2. 암묵적 전역 변수(Implicit Globals) 완전 방지
- 전역 네임스페이스를 오염시키는 암묵적 선언은 금지됩니다. 모든 변수와 종속성은 상단에 `local` 지시어를 사용하여 명시적으로 바인딩해야 합니다.
- LÖVE2D 전역 객체(`love`, `jit`, `utf8`)는 글로벌로 허용하며, `.vscode/settings.json` 내의 린터 설정에 정렬되어 있습니다.

## 3. 강력한 모듈 경계
- 큰 단일 스크립트를 생성하기보다는 각 책임 단위별(물리, 오디오, 렌더러, 객체 풀링 등)로 콤팩트한 전용 모듈을 작성합니다.
- 모듈 반환 시 항상 일관되게 `return M` 패턴을 활용하며, 상태(State)를 내부 로컬에 캡슐화하여 밖에서 부적절하게 수정하지 못하도록 보호합니다.

## 4. 결정적 업데이트 루프와 렌더링
- `update(dt)` 내에서는 시간 델타(`dt`)에 비례한 일관성 있는 상태 업데이트만 진행합니다.
- `draw()` 내부에는 화면을 그리는 렌더링 파이프라인 호출 외의 물리 상태 변경, 데이터 정렬 등의 무거운 게임 로직 연산을 엄격하게 지양합니다.

## 5. 실행 및 셀프 피드백 루프
- 코드를 변경하거나 리팩터링한 직후에는 항상 간결한 **실행 지침 및 자가 검증 방법**을 안내 메시지 끝에 제공하여 사용자가 쉽게 재현하고 확인할 수 있도록 돕습니다.
