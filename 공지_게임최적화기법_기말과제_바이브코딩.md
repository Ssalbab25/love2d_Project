# [게임최적화기법(2A)] 기말과제 안내 (바이브 코딩 프로젝트)

안녕하세요, 여러분.
이번 기말과제는 **AI 에이전트와 협업하여 하나의 프로젝트를 완성하는 경험**에 초점을 둡니다.

핵심만 먼저 안내할게요.
- 프로젝트는 **1개**를 만듭니다.
- 제출은 **게임최적화기법(2A) 과제 게시판**에 따로 합니다.
- 이 과목은 "AI를 어떻게 잘 다뤘는가"를 중심으로 평가합니다.

---

## 1. 과제 개요
- 과제 형태: 개인 또는 팀 프로젝트 (수업 운영 기준에 따름)
- 팀 제출 기준: 팀당 최대 (직접 입력)명
- 팀 제출 시 개인 기여 증빙 필수: 커밋 이력 + 역할/작업 분담표
- 개발 방식: AI 에이전트(Copilot, Claude Code, MCP 등)와 협업
- 주제: 수업 범위 안에서 자유 주제 또는 지정 주제

---

## 2. 점수 기준 (30점 만점)
1. 에이전트 환경 설계 완성도: 8점
2. 에이전트 핸들링 역량: 8점
3. 검증 및 디버깅 프로세스: 8점
4. 협업 로그 및 회고 품질: 6점

### 쉽게 말하면
- "무엇을 만들었는지"보다
- "AI를 어떻게 지시했고, 어떻게 검증하고, 어떻게 고쳤는지"를 더 중요하게 봅니다.

---

## 3. 제출 방법 (게임최적화기법(2A) 게시판)
- 제출 위치: **게임최적화기법(2A) 과제 게시판**
- 제출 형식: 압축파일 1개 + 보고서(`.md` 권장, `.txt` 허용)
- 권장 파일명: `학번_이름_게임최적화기법2A_기말.zip`
- 동일 프로젝트 제출 정책: 같은 프로젝트 산출물(코드/로그/테스트)은 과목 간 재사용 가능
- 단, 과목별 보고서 파일은 과목명에 맞게 구분해 제출

---

## 4. 제출 구조 규칙 (수업 공통 템플릿)
제출물은 수업 프로젝트와 **동일한 기본 구조**로 정리해 주세요.

필수 구조(이름/위치 유지)
- `project/src/` : 실제 실행 코드
- `project/tests/` : 테스트/검증 코드 또는 하네스
- `docs/` : 설계/요약 문서
- `worklogs/` : 작업 로그 또는 회고 기록
- `run.sh` 또는 `run.bat` : 실행 진입 스크립트(가능한 경우)
- 루트 `README.md` : 설치/실행/검증 방법

선택 구조(자유 확장)
- `assets/`, `tools/`, `scripts/`, `experiments/` 등은 자유롭게 추가 가능

구조 관련 운영 원칙
- 필수 구조가 일부 누락되면 즉시 0점 처리하지 않고, 안내 후 재제출 1회를 허용합니다.
- 재제출 이후에도 필수 구조가 불명확하면 감점될 수 있습니다.

---

## 5. 제출물 체크리스트
아래 항목을 모두 포함해 주세요.

0. 필수 구조 준수 확인
- `project/src/`, `project/tests/`, `docs/`, `worklogs/`
- 루트 `README.md`
- `run.sh` 또는 `run.bat` (해당 환경에서 제공 가능한 경우)

1. 프로젝트 원본
- 소스코드 전체
- 의존성 정보
- 실행 스크립트 또는 실행 방법

2. 에이전트 설정 자료
- 시스템 프롬프트
- 규칙 파일/설정 파일
- 커스텀 설정 파일이 없다면: 사용 도구명/버전, 기본 설정값, 실제 사용 방식 요약(1쪽 내외)으로 대체 가능

3. 에이전트 작업 로그
- CLI 로그 또는 작업 기록
- 주요 실패-수정 과정이 보이도록 정리

4. 테스트/디버깅 증빙
- 테스트 케이스
- 실행 결과
- 해결한 오류와 해결 방법

5. 게임최적화기법(2A) 보고서 (약 2쪽)
- 형식: Markdown(`.md`) 권장, TXT(`.txt`) 허용
- 파일 형식에 따른 가감점 없음 (내용 충실도와 재현 가능성 중심 평가)
- 어떤 전략으로 AI를 활용했는지
- 어떤 프롬프트/지시가 효과적이었는지
- 시행착오와 개선 포인트

---

## 6. 감점이 자주 발생하는 경우
- 로그 누락 또는 너무 축약되어 과정 추적이 어려운 경우
- "다시 만들어줘" 반복만 있고 문제 원인 분석이 없는 경우
- 결과만 제출하고 검증 기록이 없는 경우

## 7. 자주 묻는 질문
Q1. 게임알고리즘(3A)에도 같은 프로젝트를 내는데, 파일을 완전히 따로 만들어야 하나요?
- 아니요. **공통 산출물(코드/로그)은 동일하게 사용**해도 됩니다.
- 다만 게임최적화기법(2A) 게시판에는 이 과목 관점 보고서를 포함해 제출하세요.

Q2. AI가 만든 코드를 다 이해해야 하나요?
- 네. 최소한 핵심 구조와 수정 이유는 직접 설명할 수 있어야 합니다.

Q3. 실행이 완벽하지 않아도 제출 가능한가요?
- 제출은 가능하지만, 원인 분석과 개선 시도가 보고서/로그에 분명히 있어야 합니다.
