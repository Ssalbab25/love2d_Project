# 목적

LÖVE2D + Lua 프로젝트의 전체 구조를 설계하거나 개선할 때 이 스킬을 사용합니다. 모듈 경계, 의존성, 코드 재사용성을 최적화합니다.

## 절차

1. 현재 프로젝트 구조와 모듈 간 의존성을 분석합니다.
2. 단일 책임 원칙(SRP)과 모듈화 원칙을 적용하여 구조를 제안합니다.
3. update/draw 분리 원칙을 준수하는 아키텍처 패턴을 적용합니다.
4. 암묵적 전역 변수를 제거하고 명시적인 의존성 주입을 도입합니다.
5. 모듈 간 결합도를 낮추고 응집도를 높이는 리팩터링을 수행합니다.

## 아키텍처 원칙

- **모듈 분리**: 각 모듈은 명확한 책임을 가지며 독립적으로 테스트 가능해야 합니다.
- **update/draw 분리**: 상태 업데이트(update)와 렌더링(draw)을 철저히 분리합니다.
- **의존성 주입**: 모듈 간 의존성을 명시적으로 전달하여 암묵적 전역을 피합니다.
- **LuaJIT 호환**: 성능을 고려한 LuaJIT 최적화 패턴을 적용합니다.
- **게임 루프 결정성**: 숨은 프레임 부수 효과를 피하고 결정적인 동작을 보장합니다.

## 출력 형식

- 현재 구조 분석
- 제안할 아키텍처 변경사항
- 모듈 의존성 다이어그램(텍스트 기반)
- 리팩터링 단계별 계획
- 검증 방법
